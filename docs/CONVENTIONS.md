# CHALLA 협업 컨벤션

CHALLA iOS 프로젝트의 브랜치 · 커밋 · 이슈 · PR 컨벤션을 정리한 문서입니다.
개발자 3명이 협업하므로, 아래 규칙을 지켜 충돌과 혼선을 줄입니다.

---

## 1. 브랜치 컨벤션

### 형식

```
<type>/#<issue-number>-<short-description>
```

- `type` : 커밋 타입과 동일한 값 사용 (아래 타입 표 참고)
- `issue-number` : 관련 GitHub 이슈 번호
- `short-description` : 영문 소문자 + 하이픈(kebab-case)

### 예시

```
feat/#12-login-screen
fix/#31-token-refresh-bug
build/#3-tuist-designsystem-setup
refactor/#45-room-domain
```

### 규칙

- `main` 브랜치에 직접 push 하지 않습니다. 반드시 브랜치를 파고 PR로 머지합니다.
- 하나의 브랜치는 하나의 이슈에 대응합니다.
- 머지 완료된 브랜치는 삭제합니다.

---

## 2. 커밋 컨벤션

### 형식

```
[<Type>] #<issue-number> - <작업 내용>
```

### 예시

```
[Feat] #12 - 로그인 기능 구현
[Fix] #31 - 토큰 갱신 버그 수정
[Build] #3 - Tuist 디자인 시스템 모듈 세팅
```

### 타입 표

| Type | 설명 |
| :-- | :-- |
| `Feat` | **새로운 기능 추가** — 이전에 없던 새 기능이나 기능 확장 |
| `Fix` | **버그 수정** — 기존 기능이 의도대로 동작하지 않을 때 |
| `Docs` | **문서 작성** — README, 사용 설명서, 주석 등의 추가/변경 |
| `Build` | **빌드 세팅 작업** — 세팅 파일, 폴더 구조, 빌드 스크립트, 외부 종속성 변경 |
| `Refactor` | **리팩터링** — 전면 수정이 있을 때 |
| `Chore` | **그 외 잡일** — 버전 코드 수정, 패키지 구조 변경, 파일 이동/이름 변경 등 |

## 3. 이슈 컨벤션

이슈 생성 시 `CHALLA Issue Template`을 사용합니다.

### 템플릿 구조

```markdown
## 📸 About
<!-- 무엇에 관한 이슈인지 소개해주세요. -->
-

## 🍦 TO-DO LIST
<!-- 구체적인 할 일을 작성해주세요. -->
- [ ] 할일 작성

## 💬 기타 사항
```

### 규칙

- `About` : 이슈의 목적/배경을 한눈에 알 수 있게 작성합니다.
- `TO-DO LIST` : 체크박스로 구체적인 할 일을 나눕니다. (진행 상황 추적용)
- 이슈 제목은 작업 내용을 명확히 드러냅니다.

## 4. PR 컨벤션

PR 생성 시 `pull_request_template`을 사용합니다.

### 템플릿 구조

```markdown
## 📸 What is the PR?
<!-- PR에 대한 전반적인 설명을 적어주세요. -->

## 📸 Changes
<!-- 작업 내용을 리스트로 작성해주세요. -->
- ex item

## 📸 Thoughts
<!-- 깊이 고민한 내용을 작성해주세요. -->

## 📸 Screenshot
<!-- 작업한 화면이 있다면 스크린 샷으로 첨부해주세요. -->

|    기능1    |   기능2   |
| :-: | :-: |
| <스크린샷> | <스크린샷> |

## 📸 To Reviewers
<!-- 리뷰어에게 주목했으면 하는 점 or 바라는 점을 적어주세요. -->

## 📸 Related Issues
<!-- 작업한 이슈번호를 # 뒤에 붙여주세요. 이슈가 닫히는 것을 원치 않는 경우 `Resolved:`를 지워주세요 -->
- Resolved: #
```

### 규칙

- `Related Issues` : 반드시 관련 이슈 번호를 연결합니다. (`Resolved: #12`)
  - 이슈를 닫고 싶지 않으면 `Resolved:` 를 지웁니다.
- `Screenshot` : UI 작업은 스크린샷을 첨부합니다. (특히 디자인 시스템/화면 작업)
- `To Reviewers` : 리뷰어가 집중해서 봐야 할 지점을 명시합니다.
- PR 제목은 커밋 컨벤션과 동일한 형식(`[Type] #n - 내용`)을 권장합니다.
